# Recruiter Summary

Financial institutions need to identify suspicious activity while keeping decisions explainable, reviewable, and secure. This project demonstrates an end-to-end financial-crime analytics platform using only synthetic data.

It generates banking-style activity, checks data quality, builds behavioural features, trains a fraud baseline, runs AML scenarios, scores customer risk, explains model results, prepares investigation summaries, monitors platform health, and creates Power BI-ready executive data. It also provides a professional Azure architecture, security and governance design, infrastructure templates, deployment lifecycle, CI, tests, and operational runbooks.

The project is relevant to Azure data engineering, fraud/AML analytics, machine learning, MLOps, analytics engineering, risk technology, and cloud architecture roles in banks, FinTechs, payments companies, and consultancies.

It is intentionally honest: data is synthetic, the fraud model is a weak baseline, AML rules are noisy, GenAI is deterministic and local, and neither Azure nor Power BI is deployed. The value is the breadth, traceability, engineering discipline, and production-oriented reasoning demonstrated across the repository.
