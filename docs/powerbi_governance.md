# Power BI Governance

Semantic model and KPI owners should approve table grains, relationships, calculations, display formats, and business definitions. Certified metrics require reconciliation, peer review, documented limitations, endorsement, and versioned change control.

Conceptual workspaces should separate development, test, and production with deployment pipelines, refresh monitoring, rollback, and release approval. Data sensitivity labels, least privilege, row-level security by portfolio or region, object-level security for technical identifiers, periodic access review, export restrictions, and audit logging require implementation before deployment.

Refresh incidents should stop publication, identify affected tables and reports, restore a validated version, reconcile data, and document resolution. Report and model changes require impact analysis, testing, owner approval, and updated documentation.

All outputs are synthetic. No Power BI security, certification, deployment, or regulatory approval is claimed in this repository.
